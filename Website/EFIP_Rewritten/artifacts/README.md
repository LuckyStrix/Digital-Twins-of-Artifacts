# Artifact folder format

Each artifact gets its own subfolder in here. The modeling pipeline (or
you, by hand) should create one folder per artifact containing exactly:

- one model file — any of:
  - a `.glb` (preferred — everything packed into one file)
  - a loose `.gltf`, plus whatever `.bin`/texture files it references
  - a `.obj`, plus its companion `.mtl` (and textures) if it has one —
    OBJ has no materials of its own, so without a `.mtl` it loads with
    a flat default material instead of its real look
- one `.txt` file — metadata, in this format:

```
Name: Cuneiform Tablet 12
Type: tablet
Description: A clay tablet bearing an
administrative record from the
Third Dynasty of Ur.
```

Notes on the `.txt` format:

- Field order doesn't matter, and field names are case-insensitive.
- `Description:` captures everything after it through the end of the
  file. A blank line starts a new paragraph; single line breaks within
  a paragraph are treated as soft wraps and collapsed into one line.
- `Type` is freeform (e.g. `tablet`, `scroll`, `papyrus`, `other`) —
  the homepage groups artifacts by whatever value you put here.
- Folder names become the artifact's URL id, so keep them simple
  (letters, numbers, hyphens).

The folder name itself doesn't need to match the artifact name — only
the contents of the `.txt` file are shown on the site.

## Adding a new artifact

1. Create a new folder in `artifacts/` (e.g. `artifacts/tablet-014/`).
2. Drop in the model file (`.glb`, or `.gltf`/`.obj` with their
   companion files) and an `info.txt`. Any filename ending in `.txt`
   works for the metadata file.
3. From the `fully_rewritten/` folder, run:

   ```
   python3 tools/build_manifest.py
   ```

4. Reload `index.html` — the new artifact appears in its type group
   automatically, no HTML edits required.

`manifest.json` in this folder is generated by that script — don't
edit it by hand, it gets overwritten every time you rebuild.
