# Background folder format

Each background option gets its own subfolder in here, containing the
six faces of a cube map:

```
backgrounds/
  studio-grey/
    px.jpg   (+X, right)
    nx.jpg   (-X, left)
    py.jpg   (+Y, top)
    ny.jpg   (-Y, bottom)
    pz.jpg   (+Z, front)
    nz.jpg   (-Z, back)
```

- Filenames must be exactly `px`, `nx`, `py`, `ny`, `pz`, `nz` (any of
  `.jpg`, `.jpeg`, `.png`, `.webp`).
- The folder name becomes the option's id and its display name (e.g.
  `studio-grey` → "Studio Grey") — no metadata file needed.
- A folder missing any of the six faces is skipped (with a warning)
  rather than added with gaps.

## Adding a new background

1. Create a folder in `backgrounds/` and drop in the six face images.
2. From the `fully_rewritten/` folder, run:

   ```
   python3 tools/build_manifest.py
   ```

3. Reload the viewer — the new option appears in the Background panel's
   dropdown automatically.

`manifest.json` in this folder is generated by that script, same as
`artifacts/manifest.json` — don't edit it by hand.

`demo-colors/` is a placeholder (six flat colors, one per face) so the
Background panel has something to show out of the box. Delete it once
you've added real cube maps, then rerun the build script.
